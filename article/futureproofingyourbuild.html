Futureproofing your build

When building sites responsive or non-responsive, it's a good practice to use percentage widths instead of fixed widths for all elements. This is particularly important when building a non-responsive site as it allows for a quick changes to widths of elements without editing all the CSS, whilst alss allowing the transition to a responsive site to be much simpler. A good working example of this is when building a menu. Let's delve into some code - first some example HTML:

  <ul>
      <li>Item 1</li>
      <li>Item 2</li>
      <li>Item 3</li>
      <li>Item 4</li>
  </ul>

Then some example CSS:

ul {
  width: 60%;
}
.menulist li {
  width: 25%;
  text-align: center;
  float: left;
}
.menulist a {
  display: block;
  width: 100%;
}

Using these % widths means that this menu is fluid (note: NOT responsive). This means if your container is a fixed width, your menu will adjust if you need to change the width of the site. It also means that the transition to a responsive site is much simpler.
