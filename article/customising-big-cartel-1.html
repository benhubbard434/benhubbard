Customising Big Cartel - Part 1

This is now outdated - Big Cartel have updated their served scripts.

I've been working with Big Cartel for a little while now, and over a series of posts I'm going to try and surmise some basic, yet not well known facts that are key to making things work.

I'm going to focus on customising the default "Sexy" theme, so some assumptions will be made to the code that is already there. A warning to anyone who has just opened a Big Cartel account, before customising anything, take a back up of your code. Once you have pressed that save button, there is no going back.

The easiest and most common way to add a bit of personality and uniqueness to your online store is to add in jQuery plugins. In theory this should be a case of simply calling jQuery and referencing your plugin. However it is not quite that simple.

To Begin

The first thing to look at is the head section. First of all you need to pick out the line of code containing Prototype.


  <script src="prototype" type="text/javascript"></script>

To allow for any jQuery plugins to work, this has to be loaded after jQuery. The safest way to do this is to this is to ensure that it is placed right at the end of the head tag.

Next Step

The next step, and one which is commonly asked in the Big Cartel forums, is how to actually run jQuery. Due to Big Cartel using a JS to run it's cart, this causes conflict with jQuery. To avoid this, you need to run jQuery in noConflict mode.

 
 <script src="http://ajax.googleapis.com/ajax/libs/jquery/1.5.2/jquery.min.js"></script>
  
  <script type="text/javascript">
    var j = jQuery.noConflict();
  </script>

This allows both your cart/checkout and your plugins to work. Once you have done this, when calling a plugin you must make sure that you change every $ to a j

It is important to test your store fully once completing any customisation to ensure it works.
